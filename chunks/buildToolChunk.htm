Buildtools Project Structure Refrence

Refrences

all - build everything

<target name="all" depends="init,test,compile,dist">
    <echo>I'm done</echo>
</target>

compile - files are copied from the src directory to build directory concatenation, minification, adding version numbers, xsl imports

test - typically means jasmine unit testing of javascript

doc - documentation of javacsript using jsdoctoolkit

dist - files are copied to the dist folder following dctm's directory structure

deploy - not used much now, will be used later

clean - deletes cruft (build dir, doc directory, dist directory)

Stuff to commit (typical project structure): - src directory - for source code
                                                                            test directory - for unit tests, test data, etc
                                                                            README.md - explains the project, build instructions, etc
                                                                            build.xml - to tell ant what to do
